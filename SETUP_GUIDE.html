교사용 설정 가이드

EE Research Studio를 온라인으로 배포하기까지 — 처음 설정에 약 30분

1Google Sheet 준비 & Roster 입력

학생 인증 정보와 응답이 저장될 스프레드시트를 만들고, 반 명단을 입력합니다.
  1. Google Drive에서 새 스프레드시트 생성. 이름 예: EE Studio Data
  2. roster-template.html을 열어 21명의 학번(StudentID)과 이름을 정리하세요. 학번은 숫자 4~6자리를 권장합니다 (예: 학년1+반03+번호05 → 10305).
  3. 스프레드시트에 Roster라는 이름의 시트 탭을 만들고, A열에 StudentID, B열에 StudentName 헤더를 넣은 뒤 21명을 입력하세요.
  4. Students, Responses 탭은 스크립트가 처음 실행될 때 자동으로 만들어지므로 미리 만들 필요 없습니다.
Roster에 없는 학번은 사이트에서 등록/로그인이 아예 되지 않습니다 — 실제 우리 반 학생만 접속할 수 있게 하는 안전장치입니다.

2Apps Script 배포 & 교사 암호 설정

  1. 스프레드시트 메뉴에서 확장 프로그램 → Apps Script 클릭
  2. 기본 Code.gs 내용을 지우고, 제가 드린 google-apps-script/Code.gs 내용을 그대로 붙여넣기 → 저장
  3. 왼쪽 톱니바퀴 프로젝트 설정 → 맨 아래 스크립트 속성 → 속성 추가 → 속성: TEACHER_PASSCODE, 값: 선생님이 정한 암호(예: 교무실에서만 아는 문구) 입력 후 저장
    이 암호는 교사 대시보드 접속용입니다. 공개 저장소 코드에는 절대 들어가지 않고, 이 스크립트 속성에만 저장됩니다.
  4. 우측 상단 배포 → 새 배포 → 톱니바퀴 → 웹 앱 선택
  5. "실행 계정": 나, "액세스 권한": 전체(익명 사용자 포함) → 배포
  6. 권한 승인 요청 시 본인 계정으로 로그인 → "고급" → "이동" 클릭 → 허용
  7. 배포 완료 후 나오는 웹 앱 URL(.../exec로 끝남)을 복사

3사이트에 연결 & GitHub Pages 배포

  1. assets/app.js에서 PASTE_YOUR_APPS_SCRIPT_WEB_APP_URL_HERE를 방금 복사한 URL로 교체
  2. github.com에 무료 계정 생성 (이미 있다면 생략)
  3. New repository → 이름 예: ee-studio, Public으로 생성
  4. Add file → Upload files → ee-studio 폴더 안의 모든 파일/폴더를 드래그앤드롭 → Commit
  5. Settings → Pages → Branch를 main/root로 설정 → 몇 분 후 사이트 주소 생성 (형식: https://[사용자명].github.io/ee-studio/)
이 주소 하나만 학생들에게 한 번 공유하면 됩니다. Day가 늘어나도 같은 주소를 계속 사용합니다. 파일을 수정할 때는 같은 방식으로 다시 업로드하면 1~2분 내 반영됩니다.

4학생 배포 & 로그인 방식 안내

  1. Google Classroom에 사이트 주소를 한 번만 게시 (예: 상단 고정 공지 또는 "자료" 게시물)
  2. 학생들에게 안내: "처음 접속 시 본인 학번을 입력하고, 비밀번호 칸에 원하는 숫자 4자리를 입력하면 그대로 등록됩니다. 이후에는 같은 비밀번호를 입력해야 합니다."
  3. 비밀번호를 잊은 학생이 있다면: Google Sheet의 Students 탭에서 해당 학번 행을 찾아 그 행 전체를 삭제하세요 — 다음 접속 시 새 비밀번호로 재등록됩니다.
  4. 5회 연속 오답으로 잠긴 계정(Locked = TRUE)도 같은 방법으로 행을 삭제하면 초기화됩니다.

5교사 대시보드 사용법

학생 이름을 선택해 제출한 과제를 한눈에 볼 수 있습니다.
  1. 사이트 홈 맨 아래 "교사용" 링크 클릭, 또는 직접 [사이트주소]/teacher/dashboard.html 접속
  2. 2단계에서 설정한 TEACHER_PASSCODE 입력
  3. 왼쪽에서 학생 이름 검색/클릭 → 오른쪽에 해당 학생이 제출한 모든 Day의 응답이 표시됩니다
이 페이지는 GitHub 공개 저장소에 있지만, 암호 없이는 아무 데이터도 볼 수 없습니다. 다만 링크 자체는 누구나 열 수 있으니 암호는 학생들과 공유하지 마세요.

6새 Day 추가하기

  1. 열람용 유인물: handouts/_template-view.html 복사 → 내용 채우기
  2. 과제용 폼: forms/_template-form.html 복사 → DAY/FORM_ID와 필드(data-field) 채우기 (로그인 UI는 이미 포함되어 있어 그대로 두면 됩니다)
  3. Day 페이지: days/_template-day.html 복사 → days/day2.html로 저장 후 링크 연결
  4. index.html에서 해당 Day 카드의 locked 클래스 제거, href를 실제 경로로 수정
  5. GitHub에 새 파일 업로드
디자인을 전체적으로 바꾸고 싶다면 assets/handout.css(유인물), assets/form.css(과제 폼), assets/site.css(홈/Day 목록) 중 해당 파일만 고치면 이미 만든 모든 페이지에 반영됩니다.

✓테스트 체크리스트

  1. GitHub Pages 주소로 접속 → Day 1 → 유인물 1~4번이 정상적으로 열리는지 확인
  2. 5번 "작성하기" → 실제 등록된 학번 + 아무 숫자 4자리 입력 → "확인" → 새로 등록되었다는 메시지가 뜨는지 확인
  3. 내용 입력 후 저장 → Google Sheet의 Responses 탭에 행이 생겼는지 확인
  4. 같은 학번+비밀번호로 새로고침 후 "확인" → 입력했던 내용이 자동으로 채워지는지 확인
  5. 일부러 틀린 비밀번호로 5회 시도 → 잠김 메시지가 뜨는지, Students 탭에 Locked가 TRUE로 바뀌는지 확인
  6. teacher/dashboard.html 접속 → 교사 암호 입력 → 방금 테스트한 학생이 목록에 보이고 응답이 표시되는지 확인
